== Error Code Reference

Kōtuku uses a unified set of integer error codes for all API functions.  Error codes are exposed to Tiri as global constants following the `ERR_` naming convention (e.g. `ERR_Okay`, `ERR_Failed`, `ERR_Args`).  See <<ch-error-handling>> for full error handling semantics.

[[sec-exception-threshold]]
=== Exception Threshold

Error codes are divided into two categories based on their numeric value:

Minor codes (0--8)::
Control flow indicators that do not represent failures.  These codes do not trigger exceptions when used with `check` or inside `try` blocks.

Exception codes (9 and above)::
Represent genuine errors.  When a Kōtuku API function returns one of these codes inside a `try` block, the error is automatically promoted to an exception.  The `check` statement also raises an exception for any code at or above the threshold.

The threshold boundary is `ERR_Terminate` (value 9).
